# Chipyard VLSI Workspace

This repo builds a Docker image that contains the chipyard framework, and can be used to synthesis Rocket based SoCs.
It also contains a script to explore the parameter space of the Rocket configuration.

## TLDR

```bash
make docker-build
make docker-start
python scripts/param_explore.py --case-json ./usecases/case_rocket_valid.json
```



## Docker

Build image (ensure you have a working docker system)
```bash
make docker-build
```

Start container
```bash
make docker-start
```

Run a command inside the container within the chipyard conda environment
```bash
make docker-cmd CMD="ls"
```

Other useful docker commands
```bash
make docker-status    # show container + disk usage
make docker-bash      # attach an interactive shell
make docker-reset     # delete container and restart (reapplies overlays)
make docker-stop      # stop the container
make docker-root      # copy /root from container to ./docker-root
```

### Host prerequisites (Linux)
- Docker Engine (daemon running, user in `docker` group or run with sudo)
- GNU `make`
- `bash`
- `python3` (for `scripts/param_explore.py`)


## Config Overlays

Local files under `overlay` are copied into the container root on `make docker-start`.

Put files under the full container path, for example:
```
overlay/root/chipyard/generators/chipyard/src/main/scala/config/CustomRocketConfigs.scala
overlay/root/chipyard/vlsi/sky130-rocket.yml
overlay/root/chipyard/vlsi/sky130-boom.yml
```


## Simulation (Cycle and Instruction count)

It is possible to generate the verilog files for a particular chip target :

For example, here we produce the verilog of a Boom V3 SoC:
```bash
make verilog TARGET=boom
```

The verilog files are generated inside the docker under `/root/chipyard/sims/verilator/generated-src/`

This can be used to produce the first performance metrics (cycles and instructions count) using the simulator.

For example runnning the `benchamrk` target will produce the `benchmark.${TARGET}.log` wich contains the cycle and instruction count for multiple beenchmarks:
```bash
make benchmark TARGET=boom 
```
The `mm` target will produce the `mm.${TARGET}.log` wich contains the cycle and instruction count for the mm benchmark only:
```bash
make mm TARGET=boom 
```


## Syntehsis and Place & Route (Area and Power)

In order to collect the area and power metrics, we must perform at least the synthesis.

For example,
```bash
make syn TARGET=boom
```

This command generates the `syn.${TARGET}.log` log file, which should contains an estimate of the area.

Then 

```bash
make syn_power TARGET=boom
```

will run a OpenROAD script to estimate the power.

## Targets

Possible values for TARGET are :
 - rocket (a default TinyRocket)
 - boom (a BoomV3)
 - customrocket (CustomRocketConfigs.scala generated by explore_param)
 - parametricrocket (Alternative to customrocket, less parameter but more stable)

For parametricrocket, the legacy config class is `chipyard.ParametricRocketConfig`.

## Demo

Quick smoke run (parametricrocket, single case):
```bash
make demo
```
This wraps `scripts/param_explore.py --target parametricrocket --case-json ./usecases/case_parametricrocket_valid.json`
and will call `make docker-stop || true`, `make docker-reset`, `make verilog`, `make mm`, `make syn`, `make syn_power`.


## Scripts

Run a specific configuration of the customrocket target (verilog, mm, syn, syn_power):
```bash
scripts/param_explore.py --case-json ./usecases/case_rocket_valid.json
```

Parameter exploration (random search):
```bash
scripts/param_explore.py --space overlay/root/chipyard/generators/chipyard/src/main/scala/config/param_rocket.json
```

Check constraint rules for the usecases folder:
```bash
python3 scripts/check_constraints.py
```
